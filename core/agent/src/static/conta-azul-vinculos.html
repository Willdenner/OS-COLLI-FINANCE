Una o que vem do Finance (productId, paymentMethod, forma_pagamento) aos UUIDs e campos que o Conta Azul espera nos contratos e baixas. Os mesmos dados são usados no painel FP&A em Configurações → Contratos (Lovable → Conta Azul).

Como funciona: a categoria ou código de produto no Finance define qual itens[0].id usar no Conta Azul. A forma de pagamento no Finance pode definir tipo_pagamento, id_conta_financeira e item quando você preencher as colunas. Com o Conta Azul conectado, use Carregar lista para escolher o produto/serviço e preencher o UUID automaticamente.

◇ Catálogo Conta Azul
Após o OAuth no FP&A, clique em Carregar lista (API GET /v1/produtos). Por padrão mostramos só serviços (tipo SERVIÇO no cadastro).
O mesmo endpoint lista os dois. Em Somente serviços entram linhas que não são produto/kit explícitos (inclui itens sem tipo no JSON, comum em serviços).
Algumas contas exigem filtro de status; outras aceitam vazio para listar tudo.
◎ Padrões do contrato
Usado em condicao_pagamento.id_conta_financeira quando o payload não trouxer outro.
O Conta Azul recebe valores em reais; o agente converte quando necessário.
